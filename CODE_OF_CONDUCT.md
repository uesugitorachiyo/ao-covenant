# Code Of Conduct

AO Covenant is a technical project for evidence-bound agent orchestration. The
project expects direct, respectful collaboration focused on concrete behavior,
reproducible evidence, and public safety.

## Expected Behavior

Participants are expected to:

- keep discussion technical, specific, and relevant to the project
- assume mistakes can be corrected through evidence, tests, and review
- respect boundaries around security reports, private keys, credentials,
  production evidence, and local machine details
- give actionable feedback when requesting changes
- accept maintainer decisions about scope, release timing, and public API
  stability

## Unacceptable Behavior

The project does not accept:

- harassment, threats, personal attacks, or discriminatory language
- publishing another person's private information, credentials, local paths, or
  private security material
- knowingly posting exploit details or sensitive vulnerability material outside
  the [security policy](SECURITY.md)
- repeated off-topic disruption after a maintainer asks for the discussion to
  return to project scope
- impersonation, forged evidence, or intentionally misleading reports

## Reporting Conduct Issues

Report conduct issues in the least public channel that fits the issue. For
ordinary project-maintenance issues, open a GitHub issue with a concise
description. For anything involving security material, private keys,
credentials, production evidence, or suspected vulnerabilities, use the
[security policy](SECURITY.md) instead of a public issue.

Reports should include links, timestamps, affected threads or pull requests, and
the specific behavior that needs maintainer review. Do not include secrets,
private keys, production evidence, or unnecessary personal information.

## Maintainer Response

Maintainers may edit, hide, lock, or close issues and pull requests that violate
this code. Maintainers may also ask participants to move sensitive material to a
private security channel, narrow a discussion to actionable technical points, or
stop participating in a thread.

This project is pre-1.0. Maintainers may make conservative decisions to protect
project integrity, user safety, and release trust even when the contribution is
well-intended.
